## Root Level Files

| File | Purpose |
|---|---|
| `.env` | Your actual secrets вЂ” DATABASE_URL and API keys. NEVER commit to Git. |
| `.env.example` | Template with placeholder values. Commit this so teammates know what variables are needed. |
| `.gitignore` | Tells Git to ignore `.env`, `uploads/`, `.venv/`, and `__pycache__/`. |
| `pyproject.toml` | Your direct dependencies. Edit this when adding new packages via `uv add`. |
| `uv.lock` | Auto-generated by uv. Locks exact versions of every package. Never edit manually. Commit to Git. |
| `README.md` | Project description, setup instructions, and how to run the server. |


## The Golden Rules

```
1. Routers  в†’ only handle HTTP, call services, return responses
              NEVER query the database directly

2. Services в†’ contain all business logic, call models and agents
              NEVER define API routes

3. Models   в†’ only define database table structure
              NEVER contain business logic or HTTP handling

4. Schemas  в†’ only define request and response shapes
              NEVER contain database queries or business logic

5. Agents   в†’ only run AI prompts and return structured output
              NEVER touch the database directly

6. Database в†’ only holds the connection and Base
              nothing else lives in this file
```
